---
title: "Using ECharts built-in themes in uni-echarts"
description: "Switch between the built-in dark and light themes at runtime using the theme prop, and register custom JSON themes with echarts.registerTheme."
---

ECharts ships with two built-in themes — `"dark"` and `"light"` — that you can activate by passing the theme name to the `theme` prop. For custom palettes, call `echarts.registerTheme` with a theme name and configuration object, then reference the name in the same prop. This example lets you toggle between `"dark"` and `"light"` at runtime and pair the theme with a matching background color and loading overlay.

## Complete example

<CodeGroup>

```vue index.vue (script setup)
<script lang="ts" setup>
import type { LineSeriesOption } from "echarts/charts";
import { LineChart } from "echarts/charts";
import type {
  DatasetComponentOption,
  LegendComponentOption,
  PolarComponentOption,
  TooltipComponentOption
} from "echarts/components";
import {
  DatasetComponent,
  LegendComponent,
  PolarComponent,
  TooltipComponent
} from "echarts/components";
import type { ComposeOption } from "echarts/core";
import * as echarts from "echarts/core";
import { CanvasRenderer } from "echarts/renderers";
import type { CSSProperties } from "vue";
import { GLOBAL_OPTION } from "../echarts";

type EChartsOption = ComposeOption<
  | LegendComponentOption
  | TooltipComponentOption
  | PolarComponentOption
  | DatasetComponentOption
  | LineSeriesOption
>;

echarts.use([
  LegendComponent,
  TooltipComponent,
  PolarComponent,
  DatasetComponent,
  LineChart,
  CanvasRenderer
]);

const option = ref({
  ...GLOBAL_OPTION,
  legend: {
    top: 10,
    left: "center"
  },
  tooltip: {
    trigger: "axis",
    axisPointer: { type: "cross" },
    textStyle: {
      // #ifdef MP-WEIXIN
      textShadowBlur: 1
      // #endif
    }
  },
  polar: {
    radius: "60%",
    center: ["50%", "56%"]
  },
  angleAxis: {
    type: "value",
    startAngle: 0
  },
  radiusAxis: { min: 0 },
  series: [
    {
      type: "line",
      coordinateSystem: "polar",
      showSymbol: false
    }
  ],
  dataset: {
    dimensions: ["数值"],
    source: getData()
  }
} satisfies EChartsOption);

const theme = shallowRef("dark");

const loading = shallowRef(false);

const loadingOptions = computed(() => {
  const value: Record<string, unknown> = { text: "请稍候..." };

  if (theme.value === "dark") {
    Object.assign(value, {
      color: "#ffffff",
      textColor: "#ffffff",
      maskColor: "rgba(0, 0, 0, 0.7)"
    });
  }

  return value;
});

const styles = computed(() => {
  const value: CSSProperties = {};

  if (theme.value === "dark") {
    value.backgroundColor = loading.value ? "#05040d" : "#100c2a";
  }

  return value;
});

function getData() {
  const data: [number, number][] = [];

  for (let i = 0; i <= 360; i += 1) {
    const t = (i / 180) * Math.PI;
    const r = Math.sin(2 * t) * Math.cos(2 * t);
    data.push([r, i]);
  }

  return data;
}
</script>
```

```vue index.vue (template)
<template>
  <uni-echarts
    custom-class="h-75"
    :style="styles"
    :option="option"
    :theme="theme"
    autoresize
    :loading="loading"
    :loading-options="loadingOptions"
  ></uni-echarts>
</template>
```

</CodeGroup>

## How it works

### Built-in themes

Pass `"dark"` or `"light"` to the `theme` prop. ECharts applies the corresponding palette, background, and axis styles automatically:

```vue
<uni-echarts :theme="theme" :option="option"></uni-echarts>
```

```ts
const theme = shallowRef("dark"); // or "light"
```

When `theme` changes, the component disposes the old instance and initializes a new one with the updated theme. This is handled internally — no extra work is needed on your side.

### Custom themes

Register a theme object with `echarts.registerTheme` before the component mounts, then pass the registration name as `theme`:

```ts
import myTheme from "./my-theme.json";

echarts.registerTheme("my-theme", myTheme);
```

```vue
<uni-echarts theme="my-theme" :option="option"></uni-echarts>
```

You can generate a theme JSON file from the [ECharts theme builder](https://echarts.apache.org/en/theme-builder.html).

### Adapting the loading overlay to the theme

The `"dark"` theme uses a dark canvas background, so the default white loading mask looks out of place. Use a `computed` value for `loading-options` to switch colors alongside the theme:

```ts
const loadingOptions = computed(() => {
  const value: Record<string, unknown> = { text: "Loading..." };

  if (theme.value === "dark") {
    Object.assign(value, {
      color: "#ffffff",
      textColor: "#ffffff",
      maskColor: "rgba(0, 0, 0, 0.7)"
    });
  }

  return value;
});
```

### Setting a global default theme

Call `provideEchartsTheme` inside the `setup` of a parent component (for example `App.vue`) to apply a theme to every `<uni-echarts>` instance rendered beneath it — without repeating the prop on each individual chart.

<Tabs>
  <Tab title="npm">
    ```vue App.vue
    <script setup>
    import { provideEchartsTheme } from "uni-echarts/shared";

    provideEchartsTheme("dark");
    </script>
    ```
  </Tab>
  <Tab title="uni-modules">
    ```vue App.vue
    <script setup>
    import { provideEchartsTheme } from "@/uni_modules/xiaohe-echarts";

    provideEchartsTheme("dark");
    </script>
    ```
  </Tab>
</Tabs>

<Note>
  A `theme` prop on an individual `<uni-echarts>` component takes precedence over the globally provided theme.
</Note>
